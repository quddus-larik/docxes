# DocX

DocX is a production-ready, file-system driven documentation framework built on Next.js 16. It offers a "Formula-like" customization experience, allowing you to completely tailor the look and feel without fighting the framework.

## Installation

Since DocX is a Next.js template, you typically start by cloning the repository or using a starter.

```bash
git clone https://github.com/quddus-larik/docxes.git my-docs
cd my-docs
pnpm install
```

## Project Structure

- **`content/docs`**: This is where your markdown/MDX files live.
- **`x-meta.config.ts`**: The central configuration file for your site metadata, theme, and UI components.
- **`components/`**: React components used in the application.

## Configuration

The heart of DocXes is `x-meta.config.ts`. Here you can define your site's identity, enable features, and inject custom components.

```typescript
// x-meta.config.tsx

export const XMeta = createConfig({
  ...ShadcnUI, 
  siteName: "DocXes",
  siteUrl: process.env.NEXT_PUBLIC_SITE_URL || "https://docxes.vercel.app",
  searchProvider: "flexsearch",
  theme:{
    mdx:{
      highlighter: "pretty-code",
      keepBackground: false
    }
  },
  sidebar: {
    ...ShadcnUI.sidebar,
  },
  header: Header
});

export default XMeta;


```


### API Reference
```typescript
// @/lib/configuration.tsx
const defaults: XMetaConfig = {
  siteName: "DocXes",
  description: "A dynamic documentation generator framework",
  siteUrl: "https://docxes.vercel.app/",
  documentsPath: "content/docs",
  searchProvider: "local",
  theme: {
    mdx: {
      highlighter: "pretty-code",
      theme: "github-dark",
      keepBackground: false,
    },
    cssVars: {  // [keys]: [hex | rgb() | etc]
       light: "", 
       dark: "",
       root: ""
     }
  },
  header: Header,
  modeToggle: ModeToggle,
  sidebar: { component: DocSidebar, styles: {} },
  toc: { component: DocTOC, styles: {} },
  pagination: { component: DocPagination, styles: {} },
  versions: { default: "v1" }
};


export const createConfig = (overrides: Partial<XMetaConfig> = {}): XMetaConfig => {
  return {
    ...defaults,
    ...overrides,
    theme: { ...defaults.theme, ...overrides.theme },
    sidebar: { ...defaults.sidebar, ...overrides.sidebar },
    toc: { ...defaults.toc, ...overrides.toc },
    pagination: { ...defaults.pagination, ...overrides.pagination },
    versions: { ...defaults.versions, ...overrides.versions },
  };
};
```

## Creating Documentation

Documentation is versioned and file-system based.

1.  Navigate to `content/docs/v1`.
2.  Create a new file, e.g., `my-feature.mdx`.
3.  Add frontmatter:

```mdx
---
title: My Feature
description: Explaining how my feature works.
order: 1
---

# My Feature

Write your content here using **Markdown** or React components.
```

### Folder Structure

Folders created in `content/docs/v1` automatically become sections in the sidebar. To define metadata for a folder (like its title or order), add a `main.mdx` inside that folder.

## Customization

### Theming

DocX uses a "headless" Tailwind approach. You don't need to edit CSS files to change the primary color or border radius. Just update `theme.cssVars` in the config.

```typescript
// x-meta.config.tsx
cosnt XMeta = createConfig({
  ...XMeta(any),
  theme:{
    cssVars:{
      light: "",
      dark: "",
      root: ""
    }
  }
})

```


### Search

By default, DocXes uses a local `Fuse.js` search provider. You can switch to other providers (like Flexsearch) by implementing the ` searchProvider: "flexsearch" ` in `x-meta.config.tsx`.

### Component Slots

You can inject custom content into key areas of the layout:

- **Sidebar Header/Footer**: Add logos, version switchers, or status indicators.
- **TOC Header/Footer**: Add "Edit this page" links or ad placements.

```typescript
// x-meta.config.ts
sidebarHeader: ({ version }) => <div>Version: {version}</div>
```
